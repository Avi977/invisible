<objective>
Wire `frontend/pages/dashboard.jsx` to fetch real project data from the `/api/v1/projects` endpoint created in plan 01-01. The Dashboard becomes self-fetching: on mount it calls `fetchProjects()`, shows a loading state, then renders the real data. If the fetch fails, it shows a readable error with a retry button.

Why self-fetching (instead of pushing the fetch up to `app.jsx`): `frontend/app.jsx` is the shared file consumed by all 6 sister workstreams. Editing `app.jsx` here would force a 6-way merge conflict. Keeping the fetch inside `dashboard.jsx` keeps the conflict surface exactly to what the workstream OWNS, while the existing `projects` prop (still wired in `app.jsx` from `DATA_SETS[t.dataSet].projects`) becomes the fallback path for transient errors — and the Tweaks "Mock data" toggle continues working unchanged for Focus, Terminals, Tools, Analytics which haven't been wired yet.

Purpose: Phase 1 success criteria #2, #3, #4, #5 — Dashboard renders real data, all four layouts work, action buttons route correctly with real ids, Tweaks fallback preserved for OTHER pages.

Output:
- Additive edit to `frontend/data.jsx`: new `fetchProjects()` helper.
- Edits to `frontend/pages/dashboard.jsx`: add useEffect/useState fetch logic, loading + error UI, real-data preference with fallback to `props.projects`.
- Human-verify checkpoint at the end (visual inspection of all four layouts).
