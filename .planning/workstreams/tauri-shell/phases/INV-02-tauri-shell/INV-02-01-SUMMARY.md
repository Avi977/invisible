# Phase 2 Plan 01: Tauri Shell — Summary (Tasks 1–4)

## One-liner

Tauri 2.x native shell scaffolded from scratch; 5 IPC commands wrap the
existing `invisible-*` CLI surface; system tray (Open / Hide / Quit) with
close-to-hide window behaviour mirrors `bin/invisible-app` pywebview
semantics; an SSE bridge tries `${INVISIBLE_SERVER_URL or
http://127.0.0.1:8765}/api/stream` first, falls back to polling
`/api/projects` every 5 s when the local `invisible-dashboard` returns
404 (it does — verified). All four implementation tasks committed; Task 5
(human-verify) is the next gate.

### Task 2: The 5 Tauri commands (commit `016821e`)
- `list_projects()` — preferred path: `invisible-status --json` (parses to `Vec<ProjectMeta>`). Fallback: walks `${INVISIBLE_HOME or ~/.invisible}/worktrees/*/feature/.invisible-checkpoint.json`, extracting fields with the same logic as `bin/invisible-dashboard:84-116`. Returns `Ok(vec![])` for empty/missing — never `Err`. Sort key: active runs first (`state == "RUN"`), then `updated_at` descending.
- `run_orchestrator(project, task?)` — uses the explicit closure-clone skeleton from PLAN.md Step 4 (pasted verbatim). Spawns `invisible-review <project> [--task "..."]` non-blocking; pumps `CommandEvent::{Stdout,Stderr,Terminated}` onto Tauri events `run:stdout` / `run:stderr` / `run:exit`. Returns `{pid, project}` immediately. The spawned async task outlives the function call — intentional, that's how stdout streams to the frontend.
- `kill_run(project)` — shells `invisible-review <project> --stop`. Surfaces stderr as `Err(String)` on non-zero exit.
- `tail_log(project, lines)` — **path-traversal-sanitised** via `PathBuf::file_name()` (T-INV02-02 mitigation). Clamps `lines` to `[1, 10_000]` (T-INV02-03). Returns `Ok("")` for missing files (required by CONTEXT verification Step 5 — must NOT throw).
- `status()` — preferred: `invisible-status --json`. Fallback: `invisible-ps` (used as a liveness signal) + worktree-disk snapshot. The response `source` field tags which path produced the data so the frontend can show a degraded-state indicator.
- All five commands registered in `lib.rs` via `tauri::generate_handler![...]`. Cross-platform path resolution via `invisible_home()` helper — no hardcoded `/Users/…` anywhere in source (verified by the PLAN.md regex).

### Task 3: System tray + close-to-hide (commit `688ad19`)
- `TrayIconBuilder::with_id("main")` in `.setup()` with the bundle's default window icon, tooltip "Invisible", `show_menu_on_left_click(true)`.
- Menu: three `MenuItem::with_id(app, "<id>", "<label>", true, None::<&str>)` items (Open / Hide / Quit) wrapped in `Menu::with_items`.
- `on_menu_event` dispatches: `open → win.show() + win.unminimize() + win.set_focus()`, `hide → win.hide()`, `quit → app.exit(0)`.
- `WindowEvent::CloseRequested` handler on the main webview calls `window.hide()` and `api.prevent_close()` — exactly mirrors `bin/invisible-app:264-271` (pywebview `_on_window_closing` returning `False` to prevent destroy).

### Task 4: SSE bridge + frontend wrapper (commit `83d9683`)
- `src-tauri/src/sse.rs` `run_bridge(AppHandle)`:
  - Resolves base URL: `$INVISIBLE_SERVER_URL` else `http://127.0.0.1:8765`. Token mirror: `$INVISIBLE_SERVER_TOKEN` xor `$INVISIBLE_DASHBOARD_TOKEN`.
  - `sse_loop`: GETs `{base}/api/stream` with `Accept: text/event-stream`. Returns `SseNotSupported` on 404 **or** when the response content-type is not `text/event-stream` (defends against daemons that 200-respond with HTML for unmatched routes). Per-event emits `dashboard:event` with `{source: "sse", event, id, payload}`.
  - `poll_loop`: GETs `{base}/api/projects` every 5 s. Computes a `DefaultHasher` digest of the body; emits `dashboard:event` only when the hash changes (no 5-s spam on no-op).
  - Backoff: 1s → 2s → 5s → 10s capped on transient errors. Reset to 1s on `SseNotSupported` (immediate switch to polling). Never panics.
  - Spawned in `lib.rs::run()` via `tauri::async_runtime::spawn` — the JoinHandle is intentionally dropped; the task lives until the app exits.
- `frontend-vite/src/lib/tauri.js` (54 LOC, **NEW FILE — no `.jsx` modifications**):
  - `isTauri()` runtime guard.
  - `tauriInvoke(cmd, args)` — lazy-imports `@tauri-apps/api/core.invoke`. Throws with a clear message outside the Tauri runtime so callers can handle the fallback path.
  - `subscribeDashboard(cb)` — listens for `dashboard:event` and forwards `e.payload` to `cb`; returns the unlisten handle. **No-op outside Tauri** so callers don't have to branch on `isTauri()` at every site. Defensive `try/catch` around `cb` so a throwing handler doesn't break Tauri's event loop.
- `frontend-vite/package.json` adds `@tauri-apps/api: "^2"` (resolved to `2.11.0` in `pnpm-lock.yaml`). No script changes; no other deps touched.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] Removed `src-tauri/Cargo.lock` from root `.gitignore`**
- **Found during:** Task 1, when staging the scaffold.
- **Issue:** The pre-existing root `.gitignore` had `src-tauri/Cargo.lock` listed (line 55, in the "Future UI shell" block written long before the scaffold landed). The plan explicitly mandates that `Cargo.lock` for the binary crate be **committed** — Rust convention for binaries; Phase 3's signed/notarised macOS build needs reproducible deps. Without this fix, the lockfile would have been silently dropped from the repo and Phase 3 builds wouldn't be reproducible.
- **Fix:** Replaced the line with an explanatory comment block documenting the deliberate omission, so a future contributor doesn't re-add it.
- **Files modified:** `.gitignore`
- **Commit:** `1b4742a` (same commit as Task 1 scaffold)

### Cosmetic adjustments (not deviations per se, but worth recording)

**2. [comment-only] Reworded the `invisible_home()` doc comment**
- **Found during:** Task 2 verification gate.
- **Issue:** The original comment read "NEVER hardcode `/Users/ace/.invisible`" — the literal `/Users/ace/.invisible` substring tripped the plan's anti-leak grep regex (`grep -RnE '"/Users/ace/\.invisible|/Users/[a-z]+/\.invisible' src/`), producing a false positive.
- **Fix:** Reworded to "NEVER hardcode an absolute home path" — semantically equivalent, doesn't fool the regex into thinking actual code has a hardcoded path.
- **Files modified:** `src-tauri/src/commands.rs` (one line)
- **Commit:** Folded into `016821e` (Task 2 commit; the rewrite happened before the commit landed)

### No architectural changes were needed
No Rule 4 events. All five commands fit the plan's signatures exactly. The SSE bridge's content-type sanity-check is a defensive extension (still within Rule 2 — correctness, not architectural).
